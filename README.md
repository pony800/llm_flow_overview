# LLM_Flow: AI Agent 低代码流程编排开发平台

## 一、项目概述

本项目是一款**windows**平台下面向本地化大语言模型部署的低代码开发平台，采用Python+Vue3前后端分离架构。平台通过**可视化流程编排**与**底层原理级节点控制**，实现快速搭建AI应用。核心目标是通过**流程计算与提示词工程的深度配合**，使消费级硬件上的小模型在特定任务中达到完全可用的效果。项目会持续更新,添加新的节点

### 核心优势

| 功能维度     | 本平台创新点                                     | 技术价值             |
| ------------ | ------------------------------------------------ | -------------------- |
| **部署模式** | 全流程本地化部署（后续会支持api调用）            | 支持消费级硬件运行   |
| **功能扩展** | 底层原理级节点+自定义开发                        | 灵活实现复杂业务逻辑 |
| **交互设计** | 动态运行时交互界面（自定义表单界面）             | 可以自由实现用户交互 |
| **技术依赖** | 开箱即用不依赖第三方服务,部署运行简单            |                      |
| **调试支持** | 显示编译错误信息;实时显示运行错误日志            |                      |
| **流程控制** | 提供完整的流程控制节点if-else/for/while/子流程等 | 可基本替代代码开发   |

### 目标用户场景

**AI原型验证者**: 快速验证大模型，AI-Agent功能。
**AI底层学习者**: 通过可视化节点理解Agent工作原理（所有节点功能都更加偏向底层实现,并设置有默认参数）。
**本地AI流程搭建**: 本项目节点不依赖第三方服务实现且针对本地规模的硬件进行优化。

### 功能亮点

#### 核心技术支撑

- **本地化部署**：可以实现大语言模型完全本地化开发,快速验证猜想，无需第三方云服务，同时针对本地小模型优化，通过流程编排扩展模型能力边界。
- **底层逻辑支撑**：提供多种简化大模型开发的节点，所有节点提供的功能都更加接近底层，对比市面上其他的流程搭建平台其可以直接设计到底层原理。
- **流程搭建灵活**：提供大量的默认配置以供快速搭建，同时在该平台上开发不会受到节点功能的限制,实现高度精细灵活的流程搭建，同时这些底层原理以及提供的示例流程也非常适合学习。

#### 流程编排引擎

- **代码级控制**：支持if-else/for/while/Python 脚本编辑器/子流程调用，节点高度自定义化，实现完备的逻辑表达。

- **实时操作追踪**：全链路审计日志与错误引导操作追踪 ，运行时错误精确定位到节点，支持流程中途变量注入检查。

- **动态交互界面**：内置 STDIO 节点提供用户输入输出界面编辑功能,实现运行时动态交互（支持文本框/下拉框/表格等控件）。

#### 全找开发支持

- **独创MCP优化**：专为小模型设计的工具调用协议，支持多工具动态调度（后续将兼容标准MCP）。
- **原生RAG引擎**：使用python-sqlite3实现轻量化向量数据库，单数据库文件可支持10万级别数据检索，并支持使用内存缓存进行加速（用户可配置大小）。
- **零第三方依赖**：函数调用、流程控制、知识检索等核心模块均为自主实现，且提供开发文档，使用者可自主开发自定义节点。

## 三、使用说明

**初次使用建议下载 qwen2.5 系列模型使用(本项目中默认模板及其参数配置是以qwen2.5模型为准)模型推荐放在项目根目录**

### 1.打开新的文件目录

**项目首次加载会默认打开项目根目录下的project文件夹,该文件夹里有示例流程,可以在搭建流程时参考**。
配置项目目录: 点击“**配置设置**”(主界面左侧菜单栏下方) ->选择“ **目录配置**” -> 点击**“管理”**按钮 -> 在“修改目录配置”弹窗界面中编辑对应的根目录 -> 点击“**提交**”。
重置项目目录： 点击“**配置设置**”(主界面左侧菜单栏下方) ->选择“ **目录配置**” -> 点击**“重置”**按钮。（可以将项目目录重置为初始加载的项目根目录）。
项目目录说明: 
(a)项目根目录: 整个项目工程的根目录 (注意在选择“子流程”时只能选择同一个项目目录下的项目)。
(b)模型根目录: 本地大语言模型存放位置 (本项目使用 llama.cpp 运行模型，只支持 .gguf 格式模型)。
(c)数据根目录: 存放流程项目运行时使用或保存的其他文件 (如 RAG 节点保存和读取的数据库文件等, TXT_FOR 读取的文本文件等)。

### 2.新建流程

步骤：点击“**流程管理**”(主界面左侧菜单栏第一个) -> 流程管理界面中点击下方的 **"+"**按钮添加资源文件 -> 输入”**资源名称**“和”**资源路径**“ -> 点击”**提交**“按钮-> 新建流程成功。
注意事项：
（a)点击"**流程管理**"后其会保持菜单栏选中状态，只要将鼠标移入左侧菜单栏就会自动展开 '流程管理' 界面,移出后自动关闭 '流程管理' 界面。
（b)新建流程文件必须是 .json 文件格式。
（c）点击 **"+"** 按钮前,用鼠标点击选择上面的流程可以将其路径带入到**资源路径**输入框内 (若需要在同级目录下新建流程可以使用)。
（d)**资源路径**为相对路径,提交后会自动在项目根目录下创建文件夹和文件。

### 3.流程搭建  

#### --基础操作:

**打开流程**: 点击"**流程管理**"(主界面左侧菜单栏第一个) -> 选择其中任意一个流程 ->**双击**需要打开的流程。
**添加节点**: 点击**节点选择**(主界面左侧菜单栏第二个) -> 展开的抽屉中选择需要的节点 -> 拖动到流程编辑窗口释放来添加新节点。
**删除节点**: 鼠标点击选中节点 -> 按 **'Backspace'** 键删除该节点. 或 点击节点右上角 **'...'** 选则**删除**来删除节点。
**添加连线**: 鼠标点击并按住某一个节点的**连接器**并拖动到另一节点的**连接器**来添加连线。
**删除连线**: 鼠标点击连线 -> 按 **'Backspace'** 键删除该连线. 或 点击连线中间的 **'X'** 来删除连线。
**拖动节点**: 鼠标点击并拖动节点即可将节点拖动至任意位置。
**保存流程**: 在流程编辑界面使用快捷键 **'Ctrl + S'** 保存流程. 或点击编辑界面左上方第6个图标来保存流程。
**整理流程**: 鼠标点击选中某一节点 -> 使用快捷键 **'Ctrl + D'** 整理该节点后续流程. 整理规则如下:
	a)如果只有一个子节点,则将子节点高度设置为和父节点相同, 横坐标间隔50px。
	b)如果有多个子节点,则将高度最高的子节点设置到和父节点相同,其余子节点按照其之前的相对高度排在第一个子节点下方, 横坐标间隔50px
	c)自动整理程序在遇到**锁定的节点**时会终止.点击节点右上角 **'...'** 选则**锁定**/**解锁**来设置锁定状态

#### 	--搭建说明:

a)流程中必须保留一个开始节点。
b)节点**左侧白色连接器**代表激活该节点, **右侧白色连接器**在节点运行结束时激活, **右侧红色连接器**为功能连接器会在需要时被激活。
c)节点**左侧连接器**只能连接一条线,**右侧连接器**可以连接多条线. 且只能**左侧连接器**和**右侧连接器**连接, 节点不能连接自身。
d)点击**节点选择**后其会保持菜单选中状态，只要将鼠标移入左侧菜单栏就会自动展开 '节点选择' 菜单，移出后自动关闭以方便快速添加节点。
e)鼠标悬停在**节点选择**页面的节点上可以看到节点简介。

### 4.节点编辑

**双击任意节点，可打开右侧“节点详情”编辑栏抽屉, 单击抽屉外部以关闭抽屉。**

#### 	--输入参数:

如果该节点**存在输入参数**或用户可以**自定义添加输入参数**则会在抽屉中显示输入参数模块,该模块为此节点的入参。
如果此节点允许**自定义添加输入参数**则会在输入参数模块**右上方**显示 **'+添加'** 按钮。
**参数名**: 用来**展示**参数名或**自定义**参数名,同一个节点里面**不能有同名入参**.该参数名可能会在后续节点内容中使用。
**类型**: 标注该参数是什么类型,该类型分为**基础类型**(可以选择)和**自定义类型**,该类型会影响后续参数选择。
**值**: 可以按需要选择**父流程中的参数**或**者子流程中的参数**或直接输入对应的值可以在输入选择栏进行切换。
**输入选择**: 可以**按类型**选择不同的输入模式,默认为选择**父流程中的参数**可以切换为手动填写值,其对应不同的类型有不同的输入框。
**删除**: 点击 **'...'** 选择**删除**可删除对应的参数(如果允许删除的话)。
**注释**: 点击 **'...'** 选择**注释**来针对参数编写注释,其分为**类型**与**描述**两个部分。

#### -输出参数:

如果该节点**存在返回值**或用户可以**自定义添加返回值**则会在抽屉中显示输出参数模块,该模块为此节点的入参。
如果此节点允许**自定义添加返回值**则会在输出参数模块**右上方**显示 **'+添加'** 按钮。
**参数名**: 用来**展示**参数名或**自定义**参数名,同一个节点里面**不能有同名返回值**.该参数名可能会在后续节点内容中使用。
**类型**: 标注该参数是什么类型,该类型分为**基础类型**(可以选择)和**自定义类型。**
**删除**: 点击 **'...'** 选择**删除**可删除对应的参数(如果允许删除的话)。
**注释**: 点击 **'...'** 选择**注释**来针对参数编写注释,其分为**类型**与**描述**两个部分。

#### 	--节点内容:

如果该节点有额外的内容需要进行编辑(如 if-else, stdio 节点等)则会在“**处理内容**”模块中展示,按需编辑即可。

#### --注意事项:

**输出参数**: 所有的输出参数在构建的时候会被赋予初始值(Int类型赋值为0,str类型赋值为""等等),确保定义即可使用。
**参数选择器**: 对于**输入参数**一般使用 '参数选择器' 来选择其他节点的**输出参数**作为注入值，其在运行过程中运行到该节点时会为其创建上下文运行环境并将对应的输出参数的值注入到这个变量中， 参数选择器选则的范围为 "**该节点的父节点直到开始节点连线上所有节点**(即祖先节点)**中对应这个入参类型的返回值**"。
**参数反选器**: 和参数选择器实现相同但是选择参数的范围为 "**该节点的所有子节点中对应这个入参类型的返回值**"。
**参数输入框**: 大部分参数支持切换到参数输入框来直接输入固定的值作为运行时的实际参数, 其**输入框类型与参数类型相关**。
鼠标悬停在参数名上可以快速查看参数注释。

#### --解释

a)每当**出现新的构建错误或者错误日志**时,对应的按钮会**闪烁**来提示开发者查看，点击查看之后闪烁效果消失。

	**注意**: **请确流程名称不重复,否则构建错误分组会失效**
c)运行错误: 是指在流程运行中节点代码抛出的异常会被捕获并在页面右侧弹出。
d)每次点击运行流程时,**构建错误和运行错误会被清理**以显示最新的错误信息。

## 四、重要节点介绍

​    本项目节点分为5大类，分别为FlowNode ( 流程控制模类节点 )、Operation ( 逻辑操作模类节点 )、LlamaCppNode ( 使用大模型进行生成操作类节点 )、MCP ( 大模型调用工具类节点 )和RAG ( 语义相似度匹配检索增强服务节点 )。

### 1.FlowNode ( 流程控制模类节点 )

#### 	START

**开始节点**: 作用是作为整个流程的入口,一个流程必须且**只能有一个**该节点。
可以在开始节点里面定义变量(**输出参数**)，作用类似流程的全局变量,可以在后续所有节点中选择使用。

#### 	IF_ELSE

**条件选择节点**: 根据不同的条件选择不同的分支进行流程。
双击节点打开节点编辑器,在处理内容部分可以 添加/删除 条件(输入**条件名称**和**条件**)其**条件名称**会作为分支连接器的名称。
该节点的输入参数已被注入到python上下文环境中,编写条件时可以直接当作已被定义的变量使用。
该节点会按顺序从上到下运行语句,选择第一个符合条件的分支进入,若没有则进入else分支。
**注意**: **当节执行完 if 或 else 分支之后最终会返回到该节点并继续激活该节点的'结束连接器'(节点右上角白色连接器)**。
**注意**: **条件需要使用python语法进行编写,并确保语句返回为bool类型** 。

#### 	FOR_I, FOR_LIST, FOR_DICT, FOR_WHILE, FOR_TEXT

**循环控制节点**: 流程中循环控制节点请将需要循环执行的流程连接到循环节点的**for**连接器上。
**循环节点每次循环都会跟新输出参数供后续的节点使用,并重新激活for连接器**。
**注意**: **当循环结束后最终会返回到该节点并继续激活该节点的'结束连接器'(节点右上角白色连接器),执行后续内容**。
流程控制节点分为以下5种:
**FOR_I**:  在入参中填入 **startWith**, **endWith** 进行循环,并在每一次循环后更新输出参数中的 **current**,节点逻辑如下
循环将从 **startWith** 开始到 **endWith** 为止(会自动判断俩值大小来确定是增循环还是减循环)。
**FOR_LIST**: 在入参中填入 **listData** 遍历列表.每个循环都会在输出参数**Item**中放入列表中的下一个元素。
**FOR_DICT**: 在入参中填入 **dictData** 字典.每个循环都会取出新的键值对并放入输出参数**key, value**中。
**FOR_WHILE**: 双击节点打开节点编辑器,在处理内容部分**定义输入参数并编写条件**，满足条件时会激活**for连接器**。
	注意: **条件需要使用python语法进行编写,并确保语句返回为bool类型**
	注意: 输入参数已注入到python上下文中,编写条件时可以当作已被定义的变量直接使用
**FOR_TEXT**: 对文本文档进行切分循环,在输入参数中填写**txt**(在数据根目录中选择文本文件),并填写**separators**(切分规则)。
	每一个循环会在输出参数**currentText**中放入新的切分片段。
	切分规则: 支持**按字符或字符串**进行切分,如果需要同时按照多种字符串切分,使用**空格**将其间隔开即可(如 "\\\\n enter line_end")。
	效率: 对于大文本文件采取分片读取的方式并非整个加载到内存中，可以切分超大文本文件。

### 5.RAG ( 语义相似度匹配检索增强服务节点 )

​	--为大模型提供检索增强服务, 拆分为多个节点可以对RAG流程进行精细控制
​	--基于python-sqlite3实现python原生轻量向量数据库,支持标签检索.其语义相似度匹配采用余弦相似度算法
​	--支持使用内存缓存进行检索加速. 使用numpy加速匹配算法--1秒内检索约10万条数据

#### RAG_LOADER

**检索增强服务加载数据库节点**: 加载RAG服务,每个RAG服务绑定一个RAG数据库与一个词嵌入模型(gguf格式)。
**使用方法**: 双击节点打开编辑抽屉,在入参**gguf**处选择词嵌入模型(词嵌入模型放置在模型根目录里面)。
	**模型加载和运行使用llama.cpp库,仅支持gguf格式的模型, 确保其选择的是"支持整句词嵌入的模型"**。
	入参**dbFile**选择已存在的数据库文件(数据库.db文件放置在数据根目录里面),或填写**dbPath**参数来新建一个数据库。
	新建的数据库.db文件会保存在数据根目录里(支持填写诸如 aaa/bbb/rag.db 这样带多级目录的文件名称,其会在数据根目录下创建多级目录)。
	注意: 入参**dbFile**与**dbPath**必须二选一填写(**如果填入的路径文件已经存在则读取之前的文件不会覆盖之前的文件**)。
	注意: dbPath填入的参数必须使用**".db"**尾缀。
**输入参数**: 
	gguf: 选择词嵌入模型,**仅支持gguf格式的模型**,**模型必须支持整句词嵌入。**
	dbFile: 选择数据库文件(与dbPath二选一填写)。
	dbPath: 保存数据库文件到(与dbFile二选一填写)。
	cacheSize: 使用内存缓存的大小(单位Mb), 更大的缓存可以加速更多的数据检索。
**输出参数**:
	rag: 输出加**载完成的Rag服务对象**给其他rag节点使用(可以同时给多个对象进行使用)。

	id: 需要被删除的数据id。

#### RAG_DELETE_BATCH

**从向量数据库中删除多条数据**: 从向量数据库中删除多条数据。
**使用方法**: 双击节点打开编辑抽屉,在入参**rag**中选择使用**RAG_LOADER**节点加载好的向量数据库.在入参**idList**中填写需要被删除的数据id列表。
	完成从指定数据库中删除多条数据的操作。
	注意: **删除操作不会返回结果,id存在则删除,不存在则不做任何操作。**
**输入参数**: 
	rag: 选择加载完成的向量数据库,从该库中删除数据。
	idList: 需要被删除的数据id列表。

#### RAG_SEARCH

**从向量数据库中检索数据**: 在向量数据库中查询数据,支持多种查询条件过滤。
**使用方法**: 双击节点打开编辑抽屉,在入参**rag**中选择使用**RAG_LOADER**节点加载好的向量数据库. 在入参**keyContent**中填写搜索关键句。
	完成从指定数据库中检索多条数据的操作。
	注意: **语义相似度匹配算法为余弦相似度匹配**,该算法给出的语义相似度为嵌入空间中从零点的方向指向角度偏差,而忽略距离因素。
		是更加符合直觉的语义相似度算法
	举例: "好"和"非常好"之间在空间方向都为正向词,但是在程度上可能相差很远(余弦匹配度高而位置相似度低)。	
**输入参数**: 
	rag: 选择加载完成的向量数据库,从该库中检索数据。
	keyContent: 检索关键句,**会将改句与数据库中存储数据的keyContent字段做相似度匹配**,并返回相似度高的结果。
	top_k: 返回匹配的前top_k个内容。
	top_p: 返回余弦匹配相似度不低于top_p的数据。
	targetLabels: 包含的标签列表 (**会检索包含其中中所有标签的数据**)。
	excludeLabels: 排除标签列表 (**会检索不包含其中任何一条标签的数据**)。
**输出参数**: 
	searchResult: 检索结果
		其结构为 **list<{id:Long(数据自增id), similarity:Double(匹配相似度), content:String(内容), labels:List<String>(标签列表)}>。**
